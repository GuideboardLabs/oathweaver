# Phase 20 — Make Pipeline Hardening

## Context

The first production web-app build (`Projects/pet_training_app/implementation/20260507_005823_app`) shipped broken: literal Python `dict` and `list` values in the `routes-feature` and `imports-feature` slots of `app.py`, PostgreSQL `SERIAL` syntax in the SQLite schema, missing `IF NOT EXISTS` (so `init_db()` would crash on second startup), hardcoded login credentials in an `@click` handler, and a `password_hash` seed containing the plaintext string `"password_hash_1"`.

A first-attempt fix landed before this phase doc was finalized. The second build (`20260507_021439_app`) ran in **EXTEND mode** against the prior broken build and inherited every defect verbatim — proving the implementation gaps below. EXTEND mode copies the prior build directory then "fills only new slots"; because the prior slots were already populated (with broken content), the new validators and codegen never ran. Fix 7's canon-static files (`requirements.txt`, `.gitignore`, `.env.example`) were correctly added to the scaffold, but EXTEND mode also did not copy them into the rebuilt project — instead, a new `Plumbing divergence detected` lint reported the divergence and was ignored.

Phase 20 delivers nine coordinated fixes that, together, make any web_app build either correct-by-construction or hard-failed before delivery — never silently broken — regardless of whether the run is NEW BUILD or EXTEND mode.

---

## Failure-Layer Map

The original failure crossed three independent layers. The fixes are organized to close each one:

| Layer | Failure | Fixes |
|---|---|---|
| **Routing** | `domain="general_research"` set instead of `computer_science_programming`; `continuity_score=0` (no CAG memory recalled) | Fix 6 |
| **Spec generation** | LLM collapsed concrete domain nouns ("dog training tracker, login, milestones, daily goals") into a single generic `build` entity | Fix 9 |
| **Slot fill** | `str(parsed.get(key, ""))` coerces non-string JSON values into Python repr; no per-slot code-validity check; pure-boilerplate CRUD generated by LLM rather than spec-driven | Fixes 1, 2, 4 |
| **Gating** | `route_naming` lints triggered one re-fill cycle then files were written unconditionally; `_runtime_smoke_check` was symbolic, never fired real HTTP | Fixes 3, 5 |
| **Canon hygiene** | Scaffold shipped without `requirements.txt`, `.gitignore`, `.env.example`; `app.py` ran `debug=True` and leaked exception text in 500 handler | Fix 7 |
| **Mode handling** | EXTEND mode preserved poisoned slots from prior build, bypassing the new validators and codegen | Fix 8 |

## Fix 2 — Per-Slot Code-Validity Validators

**Closes:** Lack of code-vs-data check before `write_slot`. Even valid strings can be wrong-language; this layer rejects them.

**New file:** `SourceCode/agents_make/canon/slot_validators.py`

```python
"""Per-slot validators that reject data-literals and missing required tokens."""
from __future__ import annotations
import ast
import re
from dataclasses import dataclass


@dataclass(frozen=True)
class SlotViolation:
    slot: str
    rule: str
    message: str


class SlotValidationError(ValueError):
    def __init__(self, violations: list[SlotViolation]):
        self.violations = violations
        super().__init__("; ".join(f"{v.slot}/{v.rule}: {v.message}" for v in violations))


_LITERAL_NODES = (ast.Constant, ast.List, ast.Dict, ast.Tuple, ast.Set)


def _is_data_literal_only(tree: ast.Module) -> bool:
    if not tree.body:
        return False
    return all(
        isinstance(n, ast.Expr) and isinstance(n.value, _LITERAL_NODES)
        for n in tree.body
    )


def validate_python_imports(content: str) -> list[SlotViolation]:
    if not content.strip():
        return []
    try:
        tree = ast.parse(content, mode="exec")
    except SyntaxError as exc:
        return [SlotViolation("imports-feature", "syntax_error", str(exc))]
    if _is_data_literal_only(tree):
        return [SlotViolation("imports-feature", "data_literal",
                              "slot is bare data, not code")]
    for node in tree.body:
        if not isinstance(node, (ast.Import, ast.ImportFrom)):
            return [SlotViolation("imports-feature", "non_import",
                                  f"only Import/ImportFrom allowed; got {type(node).__name__}")]
    return []


def validate_python_routes(content: str) -> list[SlotViolation]:
    if not content.strip():
        return []
    try:
        tree = ast.parse(content, mode="exec")
    except SyntaxError as exc:
        return [SlotViolation("routes-feature", "syntax_error", str(exc))]
    if _is_data_literal_only(tree):
        return [SlotViolation("routes-feature", "data_literal",
                              "slot is bare data, not code")]
    has_route = any(
        isinstance(n, ast.FunctionDef)
        and any(_is_app_decorator(d) for d in n.decorator_list)
        for n in tree.body
    )
    if not has_route:
        return [SlotViolation("routes-feature", "no_route_decorator",
                              "no @app.get/post/put/delete/route function found")]
    return []


def _is_app_decorator(d: ast.expr) -> bool:
    if isinstance(d, ast.Call):
        d = d.func
    return (
        isinstance(d, ast.Attribute)
        and isinstance(d.value, ast.Name)
        and d.value.id == "app"
        and d.attr in {"get", "post", "put", "delete", "route", "patch"}
    )


def validate_sql_tables(content: str) -> list[SlotViolation]:
    if not content.strip():
        return [SlotViolation("tables", "empty", "tables slot must not be empty")]
    if not re.search(r"\bCREATE\s+TABLE\b", content, re.IGNORECASE):
        return [SlotViolation("tables", "no_create_table", "no CREATE TABLE statement")]
    if re.search(r"\bSERIAL\b", content):
        return [SlotViolation("tables", "postgres_syntax",
                              "SERIAL is PostgreSQL; SQLite uses INTEGER PRIMARY KEY AUTOINCREMENT")]
    if re.search(r"\bVARCHAR\s*\(\s*\d+\s*\)", content):
        return [SlotViolation("tables", "varchar_unnecessary",
                              "SQLite TEXT is preferred over VARCHAR(N)")]
    # Every CREATE TABLE must use IF NOT EXISTS
    creates = re.findall(r"CREATE\s+TABLE\s+(?:IF\s+NOT\s+EXISTS\s+)?", content, re.IGNORECASE)
    if_not_exists = re.findall(r"CREATE\s+TABLE\s+IF\s+NOT\s+EXISTS", content, re.IGNORECASE)
    if len(creates) != len(if_not_exists):
        return [SlotViolation("tables", "missing_if_not_exists",
                              "every CREATE TABLE must use IF NOT EXISTS")]
    return []


_HASH_PREFIXES = ("pbkdf2:", "scrypt:", "argon2:", "$2", "$argon2", "$pbkdf2")

def validate_sql_seeds(content: str) -> list[SlotViolation]:
    if not content.strip():
        return []
    for line in content.splitlines():
        m = re.search(r"password_hash[^,)]*['\"]([^'\"]+)['\"]", line)
        if m and not any(m.group(1).startswith(p) for p in _HASH_PREFIXES):
            return [SlotViolation("seeds", "plaintext_password",
                                  f"password_hash seed not a real hash: {m.group(1)!r}")]
    return []


def validate_js_state(content: str) -> list[SlotViolation]:
    if not content.strip():
        return []
    if content.lstrip().startswith(("[", "{")):
        return [SlotViolation("state", "data_literal",
                              "slot starts with bare array/object literal")]
    if not re.search(r"\bref\s*\(|\breactive\s*\(", content):
        return [SlotViolation("state", "no_reactive_binding",
                              "state slot must declare ref() or reactive() bindings")]
    return []


def validate_js_methods(content: str) -> list[SlotViolation]:
    if not content.strip():
        return []
    if content.lstrip().startswith(("[", "{")):
        return [SlotViolation("methods", "data_literal", "slot starts with bare array/object")]
    if not re.search(r"\b(async\s+)?function\s+\w+\s*\(|=\s*(async\s*)?\(", content):
        return [SlotViolation("methods", "no_function_def",
                              "methods slot must define at least one function")]
    return []


def validate_js_computed(content: str) -> list[SlotViolation]:
    if not content.strip():
        return []
    # If the slot references identifiers that aren't declared in state, flag.
    # (Cheap heuristic — full scope tracking is out of scope.)
    if "computed(" not in content:
        return [SlotViolation("computed", "no_computed",
                              "computed slot must call computed()")]
    return []


def validate_js_on_mounted(content: str) -> list[SlotViolation]:
    if not content.strip():
        return []
    if "onMounted(" not in content:
        return [SlotViolation("on-mounted", "no_on_mounted",
                              "on-mounted slot must call onMounted()")]
    return []


def validate_html_view(content: str) -> list[SlotViolation]:
    if not content.strip():
        return []
    stripped = content.lstrip()
    if not stripped.startswith("<"):
        return [SlotViolation("view-feature", "not_html",
                              "view-feature must start with an HTML tag")]
    return []


def validate_css_feature(content: str) -> list[SlotViolation]:
    if not content.strip():
        return []
    if "{" not in content or "}" not in content:
        return [SlotViolation("feature-styles", "no_rules",
                              "feature-styles must contain at least one CSS rule block")]
    if re.search(r"#[0-9a-fA-F]{3,8}\b", content):
        return [SlotViolation("feature-styles", "raw_hex_color",
                              "raw hex colors forbidden — use var(--neu-*) tokens")]
    return []


VALIDATORS = {
    ("app.py",               "imports-feature"): validate_python_imports,
    ("app.py",               "routes-feature"):  validate_python_routes,
    ("schema.sql",           "tables"):          validate_sql_tables,
    ("schema.sql",           "seeds"):           validate_sql_seeds,
    ("static/app.js",        "state"):           validate_js_state,
    ("static/app.js",        "methods"):         validate_js_methods,
    ("static/app.js",        "computed"):        validate_js_computed,
    ("static/app.js",        "on-mounted"):      validate_js_on_mounted,
    ("templates/index.html", "view-feature"):    validate_html_view,
    ("static/styles.css",    "feature-styles"):  validate_css_feature,
}


def validate_slot(rel_path: str, slot_name: str, content: str) -> list[SlotViolation]:
    fn = VALIDATORS.get((rel_path, slot_name))
    return fn(content) if fn else []
```

## Fix 3 — Blocking Lints with Bounded Retry

**Closes:** `route_naming` and `envelope_conformance` lint violations were appended to `INTEGRATION_NOTES.md` but did not block delivery.

**File:** [SourceCode/agents_make/canon/lints.py](../SourceCode/agents_make/canon/lints.py)

Add severity classification:

```python
BLOCKING_RULES = frozenset({
    "route_naming_missing",       # split from route_naming
    "envelope_conformance",
    "slot_data_literal",          # raised by Fix 2 validators
    "slot_syntax_error",
    "runtime_route_failure",      # raised by Fix 5 smoke check
    "schema_postgres_syntax",
    "schema_missing_if_not_exists",
})

def _classify(violations: list[dict]) -> tuple[list[dict], list[dict]]:
    blocking = [v for v in violations if v["rule"] in BLOCKING_RULES]
    advisory = [v for v in violations if v["rule"] not in BLOCKING_RULES]
    return blocking, advisory
```

Split `lint_route_naming` into two rule names:
- `route_naming_missing` — spec route absent from `app.py` (blocking)
- `route_naming_unexpected` — `app.py` route absent from spec (advisory; canon ships `/api/health`)

**File:** [SourceCode/agents_make/app_pool.py](../SourceCode/agents_make/app_pool.py) lines 2187-2249

Replace the single-pass retry with a bounded loop:

```python
MAX_LINT_RETRIES = 3

class BuildFailedError(RuntimeError):
    """Raised when a build cannot be made clean within retry budget."""

for attempt in range(MAX_LINT_RETRIES):
    violations = run_policy_lints(
        app_py=app_py_text, app_js=app_js_text,
        feature_styles=feature_styles, spec=spec,
    )
    blocking, advisory = _classify(violations)
    if not blocking:
        break
    issue_notes = _format_violations(blocking)
    imports_slot, routes_slot, _ = _step_api_slot_fill(
        client, ..., issue_notes=issue_notes
    )
    write_slot(slot_files["app_py"], "imports-feature", imports_slot)
    write_slot(slot_files["app_py"], "routes-feature", routes_slot)
    app_py_text = slot_files["app_py"].read_text(encoding="utf-8")
else:
    raise BuildFailedError(
        f"Build failed after {MAX_LINT_RETRIES} retries:\n{_format_violations(blocking)}"
    )
```

`BuildFailedError` propagates up through `app_pool.run`. The orchestrator catches it at the boundary, writes `BUILD_FAILED.md` to the project dir with the full violation list, and surfaces the failure to the chat layer instead of delivering a broken build.

## Fix 4 — Deterministic CRUD + Schema Codegen

**Closes:** LLM-generated boilerplate is the highest-failure-rate slot. Replace with templates driven directly by `AppSpec`.

**New file:** `SourceCode/agents_make/canon/codegen.py`

```python
"""Deterministic code generation from AppSpec.

Removes pure-boilerplate slots from LLM responsibility. The LLM keeps
responsibility for non-canonical logic (auth, custom endpoints) only.
"""
from __future__ import annotations
from .app_spec import AppSpec, Entity, Route

PY_TYPE_TO_SQL = {
    "str": "TEXT", "int": "INTEGER", "float": "REAL", "bool": "INTEGER",
    "date": "DATE", "datetime": "TIMESTAMP", "json": "TEXT",
}

PY_TYPE_TO_SAMPLE = {
    "str": "test", "int": 1, "float": 1.0, "bool": True,
    "date": "2026-01-01", "datetime": "2026-01-01T00:00:00", "json": {},
}


def render_imports(spec: AppSpec) -> str:
    needs_hash = any(
        any(f.name in {"password_hash", "password"} for f in e.fields)
        for e in spec.entities
    )
    needs_datetime = any(
        any(f.type in {"date", "datetime"} for f in e.fields)
        for e in spec.entities
    )
    lines: list[str] = []
    if needs_hash:
        lines.append(
            "from werkzeug.security import generate_password_hash, check_password_hash"
        )
    if needs_datetime:
        lines.append("from datetime import datetime, date")
    return "\n".join(lines)


def render_schema(spec: AppSpec) -> tuple[str, str]:
    """Return (tables_slot, seeds_slot)."""
    tables: list[str] = []
    for entity in spec.entities:
        cols = ["    id INTEGER PRIMARY KEY AUTOINCREMENT"]
        for f in entity.fields:
            sql_type = PY_TYPE_TO_SQL.get(f.type, "TEXT")
            parts = [f"    {f.name} {sql_type}"]
            if f.required and f.default is None:
                parts.append("NOT NULL")
            if f.type == "datetime" and f.name in {"created_at", "updated_at"}:
                parts.append("DEFAULT CURRENT_TIMESTAMP")
            elif f.default is not None:
                parts.append(f"DEFAULT {_sql_default(f.default, f.type)}")
            cols.append(" ".join(parts))
        tables.append(
            f"CREATE TABLE IF NOT EXISTS {entity.name}s (\n"
            + ",\n".join(cols) + "\n);"
        )
    return "\n\n".join(tables), ""


def _sql_default(value, type_: str) -> str:
    if type_ in ("int", "float", "bool"):
        return str(int(value)) if type_ == "bool" else str(value)
    return f"'{value}'"


def render_routes(spec: AppSpec) -> str:
    by_entity = {e.name: e for e in spec.entities}
    blocks: list[str] = []
    for route in spec.routes:
        entity = by_entity.get(route.entity) if route.entity else None
        if entity is None:
            continue  # non-entity routes filled by LLM in routes-extra slot
        block = _render_handler(entity, route)
        if block:
            blocks.append(block)
    return "\n\n\n".join(blocks)


def _render_handler(entity: Entity, route: Route) -> str:
    table = f"{entity.name}s"
    is_collection = "<int:id>" not in route.path
    handler = {
        ("GET",    True):  _render_list,
        ("GET",    False): _render_get,
        ("POST",   True):  _render_create,
        ("PUT",    False): _render_update,
        ("DELETE", False): _render_delete,
    }.get((route.method, is_collection))
    return handler(entity, route, table) if handler else ""


def _render_list(entity, route, table):
    return (
        f"@app.get(\"{route.path}\")\n"
        f"def {route.handler_name}():\n"
        f"    \"\"\"List all {table}.\"\"\"\n"
        f"    db = get_db()\n"
        f"    rows = db.execute(\"SELECT * FROM {table} ORDER BY id DESC\").fetchall()\n"
        f"    return ok_items([row_to_dict(r) for r in rows])"
    )


def _render_get(entity, route, table):
    return (
        f"@app.get(\"{route.path}\")\n"
        f"def {route.handler_name}(id):\n"
        f"    \"\"\"Get a single {entity.name}.\"\"\"\n"
        f"    db = get_db()\n"
        f"    row = db.execute(\"SELECT * FROM {table} WHERE id = ?\", (id,)).fetchone()\n"
        f"    if row is None:\n"
        f"        return err(\"NOT_FOUND\", \"{entity.name} not found\", status=404)\n"
        f"    return ok_item(row_to_dict(row))"
    )


def _render_create(entity, route, table):
    writable = [f for f in entity.fields if f.name not in {"id", "created_at", "updated_at"}]
    field_names = [f.name for f in writable]
    cols = ", ".join(field_names)
    placeholders = ", ".join("?" for _ in field_names)
    body_extracts = "\n".join(
        f"    {f.name} = body.get({f.name!r})" for f in writable
    )
    required_checks = "\n".join(
        f"    if {f.name} is None:\n"
        f"        return err(\"BAD_REQUEST\", \"{f.name} is required\", status=400)"
        for f in writable if f.required
    )
    values_tuple = ", ".join(field_names)
    return (
        f"@app.post(\"{route.path}\")\n"
        f"def {route.handler_name}():\n"
        f"    \"\"\"Create a new {entity.name}.\"\"\"\n"
        f"    body = request.get_json(silent=True) or {{}}\n"
        f"{body_extracts}\n"
        f"{required_checks}\n"
        f"    db = get_db()\n"
        f"    cur = db.execute(\n"
        f"        \"INSERT INTO {table} ({cols}) VALUES ({placeholders})\",\n"
        f"        ({values_tuple},),\n"
        f"    )\n"
        f"    db.commit()\n"
        f"    row = db.execute(\"SELECT * FROM {table} WHERE id = ?\", (cur.lastrowid,)).fetchone()\n"
        f"    return ok_item(row_to_dict(row), status=201)"
    )


def _render_update(entity, route, table):
    writable = [f for f in entity.fields if f.name not in {"id", "created_at"}]
    set_clauses = ", ".join(f"{f.name} = ?" for f in writable)
    values_extract = ", ".join(f"body.get({f.name!r})" for f in writable)
    return (
        f"@app.put(\"{route.path}\")\n"
        f"def {route.handler_name}(id):\n"
        f"    \"\"\"Update a {entity.name}.\"\"\"\n"
        f"    body = request.get_json(silent=True) or {{}}\n"
        f"    db = get_db()\n"
        f"    cur = db.execute(\n"
        f"        \"UPDATE {table} SET {set_clauses} WHERE id = ?\",\n"
        f"        ({values_extract}, id),\n"
        f"    )\n"
        f"    if cur.rowcount == 0:\n"
        f"        return err(\"NOT_FOUND\", \"{entity.name} not found\", status=404)\n"
        f"    db.commit()\n"
        f"    row = db.execute(\"SELECT * FROM {table} WHERE id = ?\", (id,)).fetchone()\n"
        f"    return ok_item(row_to_dict(row))"
    )


def _render_delete(entity, route, table):
    return (
        f"@app.delete(\"{route.path}\")\n"
        f"def {route.handler_name}(id):\n"
        f"    \"\"\"Delete a {entity.name}.\"\"\"\n"
        f"    db = get_db()\n"
        f"    cur = db.execute(\"DELETE FROM {table} WHERE id = ?\", (id,))\n"
        f"    if cur.rowcount == 0:\n"
```

## Fix 5 — Real Runtime Verification

**Closes:** `_runtime_smoke_check` was symbolic. Replace with subprocess + real HTTP probes.

**File:** [SourceCode/agents_make/app_pool.py](../SourceCode/agents_make/app_pool.py)

```python
import json
import os
import socket
import subprocess

def _runtime_smoke_check(app_dir: Path, spec: AppSpec) -> list[dict]:
    port = _ephemeral_port()
    env = {**os.environ, "FLASK_DEBUG": "0", "PORT": str(port)}
    proc = subprocess.Popen(
        [sys.executable, str(app_dir / "app.py")],
        cwd=str(app_dir), env=env,
        stdout=subprocess.PIPE, stderr=subprocess.PIPE,
    )
    failures: list[dict] = []
    try:
        if not _wait_for_health(port, timeout=8.0):
            stderr = proc.stderr.read(2048).decode("utf-8", "replace")
            return [{"rule": "runtime_route_failure", "route": "/api/health",
                     "status": "no_response", "stderr": stderr}]
        for route in spec.routes:
            failures.extend(_probe_route(port, route, spec))
    finally:
        proc.terminate()
        try:
            proc.wait(timeout=3)
        except subprocess.TimeoutExpired:
            proc.kill()
    return failures


def _ephemeral_port() -> int:
    with socket.socket() as s:
        s.bind(("127.0.0.1", 0))
        return s.getsockname()[1]


def _wait_for_health(port: int, timeout: float) -> bool:
    deadline = time.monotonic() + timeout
    while time.monotonic() < deadline:
        try:
            r = urllib.request.urlopen(
                f"http://127.0.0.1:{port}/api/health", timeout=0.5
            )
            if r.status == 200:
                return True
        except (urllib.error.URLError, ConnectionError):
            time.sleep(0.2)
    return False


def _probe_route(port: int, route: Route, spec: AppSpec) -> list[dict]:
    url_path = route.path.replace("<int:id>", "1")
    url = f"http://127.0.0.1:{port}{url_path}"
    body = _synth_body(route, spec) if route.method in {"POST", "PUT"} else None
    req = urllib.request.Request(
        url, method=route.method,
        data=json.dumps(body).encode("utf-8") if body else None,
        headers={"Content-Type": "application/json"} if body else {},
    )
    try:
        r = urllib.request.urlopen(req, timeout=3.0)
        if r.status >= 400:
            return [{"rule": "runtime_route_failure", "route": route.path,
                     "method": route.method, "status": r.status}]
    except urllib.error.HTTPError as e:
        # 404 on GET /<id>=1 with empty seed table is acceptable; flag others.
        if e.code in (404,) and route.method == "GET" and "<int:id>" in route.path:
            return []
        return [{"rule": "runtime_route_failure", "route": route.path,
                 "method": route.method, "status": e.code}]
    except Exception as exc:
        return [{"rule": "runtime_route_failure", "route": route.path,
                 "method": route.method, "status": "error", "error": str(exc)}]
    return []


def _synth_body(route: Route, spec: AppSpec) -> dict:
    entity = next((e for e in spec.entities if e.name == route.entity), None)
    if entity is None:
        return {}
    return {
        f.name: PY_TYPE_TO_SAMPLE[f.type]
        for f in entity.fields
        if f.required and f.name != "id"
    }
```

## Fix 8 — Extend-Mode Slot Revalidation

**Closes:** EXTEND mode preserved poisoned slots from prior builds, bypassing all of Fixes 1–5.

**Background:** When the user requests a build for a project that already has prior implementations, `app_pool` enters EXTEND mode: it copies the prior build directory and only fills "new" slots. Inherited slots are accepted as-is. This is fundamentally incompatible with adding new validators after the fact — every prior-broken build poisons all future EXTEND builds in that project.

**Two-part fix:**

### 8a — Canon-version forced rescaffold

When entering EXTEND mode, compare the prior build's `.canon-version` against the current canon. If they differ, treat the build as NEW BUILD against the same project: copy fresh scaffold, run all slot-fill steps, but seed the LLM with `prior_build_summary` as context so the rebuild preserves the user's intent.

```python
def _enter_extend_mode(prior_build: Path, app_dir: Path, spec: AppSpec) -> str:
    """Return mode: 'extend' or 'rescaffold'."""
    prior_version = (prior_build / ".canon-version").read_text().strip() if (prior_build / ".canon-version").exists() else ""
    current_version = (CANON_ROOT / ".canon-version").read_text().strip()
    if prior_version != current_version:
        # Canon has evolved — full rescaffold with prior context as hint
        copy_scaffold(CANON_VERSION, app_dir)
        return "rescaffold"
    # Same canon version — copy prior build, then revalidate slots
    shutil.copytree(prior_build, app_dir, dirs_exist_ok=True)
    return "extend"
```

### 8b — Slot revalidation on inherited builds

Even when canon versions match, every inherited slot must pass Fix 2's validators before being treated as "already filled." Slots that fail validation are cleared — the normal slot-fill pipeline then re-fills them via the new path (deterministic codegen for routes/schema; validated LLM for the rest).

```python
def _revalidate_inherited_slots(app_dir: Path) -> list[str]:
    """Clear any slot whose content fails validation. Return list of cleared slots."""
    from .canon.slot_validators import VALIDATORS, validate_slot
    cleared: list[str] = []
    for (rel_path, slot_name) in VALIDATORS.keys():
        file_path = app_dir / rel_path
        if not file_path.exists():
            continue
        content = read_slot(file_path, slot_name)
        violations = validate_slot(rel_path, slot_name, content)
        if violations:
            write_slot(file_path, slot_name, "", validate=False)  # clear
            cleared.append(f"{rel_path}/{slot_name}")
    return cleared
```

Call `_revalidate_inherited_slots(app_dir)` immediately after the EXTEND copy, before any `_step_*_slot_fill` reads "existing slot content" as fallback.

**Plumbing reconciliation:** also in extend mode, copy any canon static files (`requirements.txt`, `.gitignore`, `.env.example`) that are present in canon but missing in the build dir. The current `Plumbing divergence detected` lint becomes redundant — replaced with active reconciliation.

```python
def _reconcile_plumbing(app_dir: Path) -> list[str]:
    """Copy canon static files that are missing in the build dir."""
    canon = CANON_ROOT / "web_app_v1"
    static_files = ["requirements.txt", ".gitignore", ".env.example"]
    copied: list[str] = []
    for name in static_files:
        src = canon / name
        dst = app_dir / name
        if src.exists() and not dst.exists():
            shutil.copyfile(src, dst)
            copied.append(name)
    return copied
```

## Fix 9 — Spec-Generator Concreteness Gate

**Closes:** Meta-prompts ("Build the true MVP of this application with all context for this domain, topic and thread contents") produce specs where every "feature" is a pipeline-internal term.

**File:** [SourceCode/agents_make/app_pool.py](../SourceCode/agents_make/app_pool.py) — extend `_step_spec_generator`:

```python
META_TERMS = frozenset({
    "mvp", "application", "app", "system", "context", "domain", "topic",
    "thread", "build", "feature", "requirement", "spec", "implementation",
    "tracker", "manager", "tool", "platform", "service", "module", "page",
})

GENERIC_ENTITY_NAMES = frozenset({
    "build", "item", "thing", "entry", "record", "object", "data", "info",
})


class SpecConcretenessFailure(ValueError):
    def __init__(self, issues: list[str], original_request: str):
        self.issues = issues
        self.original_request = original_request
        super().__init__("; ".join(issues))


def _validate_spec_concreteness(spec: AppSpec, original_request: str) -> list[str]:
    issues: list[str] = []
    summary_tokens = re.findall(r"\b\w{4,}\b", spec.feature_summary.lower())
    concrete = [t for t in summary_tokens if t not in META_TERMS]
    if len(concrete) < 3:
        issues.append(
            f"feature_summary too abstract: only {len(concrete)} concrete tokens "
            f"({concrete!r})"
        )
    if len(spec.entities) == 1 and spec.entities[0].name in GENERIC_ENTITY_NAMES:
        issues.append(
            f"single generic entity '{spec.entities[0].name}' — no domain noun extracted"
        )
    if not spec.entities:
        issues.append("spec has zero entities")
    return issues


def _step_spec_generator(client, original_request: str, ...) -> AppSpec:
    spec = _llm_generate_spec(client, original_request, ...)
    issues = _validate_spec_concreteness(spec, original_request)
    if issues:
        # Single retry with a sharper prompt
        spec = _llm_generate_spec(
            client, original_request,
            extra_instruction=(
                "Previous attempt was too abstract. "
                f"Issues: {issues}. "
                "Extract concrete domain nouns from the request. "
                "Each entity must correspond to a real-world thing the user mentioned. "
                "Refuse to use generic names like 'build', 'item', 'thing'."
            ),
        )
        issues = _validate_spec_concreteness(spec, original_request)
        if issues:
            raise SpecConcretenessFailure(issues, original_request)
    return spec
```

`SpecConcretenessFailure` propagates up through the orchestrator. Instead of attempting the build, the orchestrator surfaces a clarifying question to the chat layer:

> "I couldn't find concrete features in your request. Could you describe what the app should do in terms of what users see and do? For example: 'Users log in. Each user has [things]. Each [thing] has [fields].'"

The user's reply is fed into a fresh `_step_spec_generator` call with the original request preserved as additional context.

Each step is independently mergeable and unit-testable. Land in this order — earlier fixes are dependencies for later ones to be testable end-to-end.

| Order | Fix | Effort | Why this position |
|---|---|---|---|
| 1 | **Fix 7** — Canon scaffold hygiene + version bump | 30 min | Bumps `.canon-version`; everything that follows depends on the version delta to trigger Fix 8a rescaffold |
| 2 | **Fix 1** — Strict slot extractor | 30 min | Lowest risk; closes the literal-data path immediately |
| 3 | **Fix 2** — Slot validators | 2-3 hr | Defense-in-depth on top of Fix 1; shared dependency for Fixes 3 and 8 |
| 4 | **Fix 8** — Extend-mode revalidation + reconciliation | 2 hr | Without this, every prior-broken build keeps poisoning new builds. Must land before Fix 4 has any effect on EXTEND projects |
| 5 | **Fix 6** — Domain override + CAG seed | 1 hr | Restores upstream context so the spec generator has something to ground in |
| 6 | **Fix 9** — Spec concreteness gate | 1 hr | Catches the next-most-common failure mode (degenerate spec) before it reaches the build pipeline |
| 7 | **Fix 3** — Blocking lints + retry loop | 1 hr | Now that lints are reliable signal, gate on them |
| 8 | **Fix 4** — Deterministic CRUD + schema codegen | 4-6 hr | Largest change; replaces the highest-risk slots. Requires Fix 2 for validation and Fix 7 for werkzeug dependency |
| 9 | **Fix 5** — Real runtime verification | 3-4 hr | End-to-end gate; depends on Fix 4 producing valid routes |

**Estimated total:** ~2 days of focused work.

**Land sequence on a single branch.** Do not ship Fix 7 + 8 without Fixes 1-4 in the same release — partial landing is what produced attempt #1's failure mode (canon evolved but build pipeline didn't catch up).

## Anti-Patterns That Defeated Attempt #1

These are explicit DO-NOTs informed by the failed first implementation pass.

1. **Don't bump canon files without bumping `.canon-version`.** Adding `requirements.txt` to the canon scaffold without the version bump means existing builds never get the new files (EXTEND mode same-version path).
2. **Don't add validators without wiring them into `write_slot`.** A validator file in the repo that's never called is dead weight. Verify the wire-in by running a test that triggers a known-bad slot.
3. **Don't add lints without classifying severity.** A new `Plumbing divergence detected` lint that doesn't gate delivery is exactly the failure mode of `route_naming` from attempt #1.
4. **Don't test fixes by re-running the same project name.** EXTEND mode is the trap. Always test in a fresh `Projects/<new-name>/` directory until Fix 8 is verified working.
5. **Don't trust `py_compile` to catch slot poisoning.** Every Python data literal compiles cleanly. The catch must happen at the AST-shape level (Fix 2) or at runtime via real HTTP probes (Fix 5).
6. **Don't preserve broken slots as "existing content" fallback.** The current `_step_*_slot_fill` functions use existing slot content as fallback when the LLM fails. After Fix 8 clears poisoned slots, this fallback path is safe — but only after Fix 8 lands.
